1. 文件夹内容
assets存的是要用的图片
cloud云函数，不用管
pages/
	comment:评价的首页，就是展示有哪些食堂，点击每个食堂可以进入每个食堂的详细页面
	detail:每个食堂的详细页面，里面有价格、评分、食堂照片、评论
	detect_num/
		home:展示所有食堂的人流量信息
		time:展示某个食堂的人流量信息
	index:首页
	info:提交评论、打分的页面
	logs:没用，但是好像不能删除，不用管
	userInfo:我的页面
styles:weui的样式库

2.框架逻辑：
首页展示三个模式：
	省时模式：点击之后进入pages/detect_num/home，展示每个食堂的人数（用小人表示）(如果能在小人上面加上食堂人数数据更好），
		再点击每个食堂进入page/detect_num/time，通过传递参数判别是哪个餐厅（1代表餐厅1，以此类推......）点击“获取食堂实时情况”
		得到当前食堂照片（未处理）、点检测食堂人流量对当前照片进行分析，返回处理后的照片和人数
	经济模式、美食模式：点击之后之间展示弹框（简便），点击详细信息进入这个食堂的详细页面/pages/detail（传参判断是哪个食堂）

tabbar里三个模块：
	首页
	餐厅：展示所有餐厅的信息，点击某个餐厅后跳到餐厅详细页面；还有提交用餐评价的按钮，点击进入/pages/info，里面可以提交价格、
		评分、评论。
	我的：展示一些个人信息
